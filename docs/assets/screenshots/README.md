# Product screenshot policy

Only screenshots captured from a purpose-built demo Project may be committed.
Before adding an image, verify that it contains none of the following:

- browser tabs, bookmarks, account names, or notifications;
- real absolute paths, usernames, repositories, or Project names;
- credentials, configuration secrets, prompts, filenames, or file contents;
- provider account details or private Session history;
- temporary-path names or debug overlays.

Use a 16:10 viewport, the default dark theme, and synthetic content that
demonstrates one focused workflow. Prefer WebP and keep each image below
500 KB. Record the Kubecode commit used to capture the image in the pull request.

The current README uses the brand preview until a clean demo workspace can be
captured through the supported browser workflow. Do not substitute personal
screenshots.
